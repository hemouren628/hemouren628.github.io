leetcode面试经典一百五十题：

加油站

在一条环路上有 n 个加油站，其中第 i 个加油站有汽油 gas[i] 升。

你有一辆油箱容量无限的的汽车，从第 i 个加油站开往第 i+1 个加油站需要消耗汽油 cost[i] 升。你从其中的一个加油站出发，开始时油箱为空。

给定两个整数数组 gas 和 cost ，如果你可以按顺序绕环路行驶一周，则返回出发时加油站的编号，否则返回 -1 。如果存在解，则 保证 它是 唯一 的。

示例 1:

示例 2:

提示:

  • gas.length == n
  • cost.length == n
  • 1 <= n <= 105
  • 0 <= gas[i], cost[i] <= 104

题解

本题的一个思路是确定一个点，从i节点最远能跑到j节点（数组需要看成圈循环，利用除余），那么i-j这中间的任意一点都不会超过j，因为中间任意一点剩余油量必须>=0才能继续往下开，所以根据这一点，开始遍历数组，找到i点能到达的最远位置，如果这个位置到了起点i，则返回此时的下标，如果没到，则从j+1开始往后找，因为中间不可能有跑更远的，如果在找最远位置的时候回到了i前面，就说明后面的点都达不到，可以直接返回-1；

leetcode面试经典一百五十题：

除自身以外数组的乘积

给你一个整数数组 nums，返回 数组 answer ，其中 answer[i] 等于 nums 中除 nums[i] 之外其余各元素的乘积 。

题目数据 保证 数组 nums之中任意元素的全部前缀元素和后缀的乘积都在 32 位 整数范围内。

请 不要使用除法，且在 O(n) 时间复杂度内完成此题。

示例 1:

示例 2:

提示：

  • 2 <= nums.length <= 105
  • -30 <= nums[i] <= 30
  • 保证 数组 nums之中任意元素的全部前缀元素和后缀的乘积都在 32 位 整数范围内

题解

看到本题第一个想到全部乘起来再除nums[i],结果不让使用除法，不然就变成简单题了

思路：除自身以外其他数乘起来可以看成两部分，前面所有数乘起来乘上后面所有数，照这个思路，我们可以定义两个数组，分别存储nums[i]前面所有数的乘积，和后面所有数的乘积，然后他们的乘积就是不包括其本身的所有乘积

leetcode面试经典一百五十题：

H 指数

给你一个整数数组 citations ，其中 citations[i] 表示研究者的第 i 篇论文被引用的次数。计算并返回该研究者的 h 指数。

根据维基百科上 h 指数的定义：h 代表“高引用次数” ，一名科研人员的 h 指数 是指他（她）至少发表了 h 篇论文，并且 至少 有 h 篇论文被引用次数大于等于 h 。如果 h 有多种可能的值，**h 指数** 是其中最大的那个。

示例 1：

示例 2：

提示：

  • n == citations.length
  • 1 <= n <= 5000
  • 0 <= citations[i] <= 1000

题解

在本题的三个h中，显然第一个h不用管，就是数组长度且一定小于第二个h，题目就是求一个数组中，有x个数都大于等于x，x的最大值。有点绕？举例说明

[1,2,3,4,5]在此数组中，大于等于1的有5个，大于等于2的有4个，大于等于3的有3个，大于等于4的有2个，大于等于5的有1个，所以3符合。

[0,1,1,5,6]在此数组中，大于等于0的有5个，大于等于1的有4个，大于等于5的有2个，大于等于6的有1个，所以h为2。

看到规律了吗？在上面两个数组中我都选择了有序的数组，我们可以观察到，排序后，数组的数字在增加，所以大于等于其的数就在减少，只要排序后循环一遍，找到第一次数字大于等于后面剩余数字的个数，就是h的最大值。
